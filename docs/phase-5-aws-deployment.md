**Goal:** Deploy to AWS with auto-scaling, HTTPS, and CI/CD at www.instnews.net

## Architecture

```
    Route 53 (instnews.net)
            │

            │
```

## Deployment Steps

### Prerequisites (Manual — One Time)

1. **AWS Account** with billing configured
2. **AWS CLI** installed and configured (`aws configure`)
3. **Domain** — instnews.net nameservers pointing to Route 53
4. **GitHub Secrets** configured:

| Secret | Value | Where to get |
|--------|-------|--------------|
| `AWS_ROLE_ARN` | `arn:aws:iam::ACCOUNT:role/github-deploy` | IAM → Create OIDC role for GitHub Actions |

### Step 1: Create Route 53 Hosted Zone

```bash
# If domain is not already on Route 53
aws route53 create-hosted-zone --name instnews.net --caller-reference $(date +%s)
# Then update domain registrar to use the NS records from the hosted zone
```

### Step 2: Deploy CDK Stack

```bash
cd infra
pip install -r requirements.txt

# First time: bootstrap CDK in your account
cdk bootstrap aws://ACCOUNT_ID/us-east-1

# Review what will be created
cdk diff

# Deploy
cdk deploy --context domain=instnews.net
```

This creates: VPC, RDS, ECS cluster, ALB, ACM certificate, ECR repo, Route 53 records, Secrets Manager entries.

**CDK will output:**
- `AlbDns` — ALB DNS name
- `EcrRepoUri` — ECR repository URI (needed for first image push)
- `DbEndpoint` — RDS endpoint
- `WebUrl` — https://www.instnews.net

### Step 3: Push First Docker Image

```bash
# Get ECR login
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com

# Build and push
docker build -f Dockerfile.prod -t ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com/instantnews:latest .
docker push ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com/instantnews:latest
```

### Step 4: Configure Secrets

Update the `instantnews/app` secret in Secrets Manager with:
```json
{
  "STRIPE_SECRET_KEY": "sk_...",
  "STRIPE_WEBHOOK_SECRET": "whsec_...",
  "STRIPE_PRICE_PLUS": "price_...",
  "STRIPE_PRICE_MAX": "price_...",
  "FIREBASE_CREDENTIALS_JSON": "{...service account JSON...}"
}
```

The `instantnews/db` secret is auto-generated by CDK and contains `DATABASE_URL`.

### Step 5: Verify

```bash
# Check ECS services are running
aws ecs list-services --cluster instantnews

# Check ALB health
curl -s https://www.instnews.net/api/stats | jq .

# Check logs
aws logs tail /ecs/instantnews-web --follow
aws logs tail /ecs/instantnews-worker --follow
```

## Deployment

### Primary: Deploy Script (platform-independent)

```bash
./deploy.sh              # Full deploy: test → build → push → deploy
./deploy.sh --skip-tests # Skip tests (faster)
./deploy.sh --web-only   # Only redeploy web service
./deploy.sh --worker-only # Only redeploy worker
```

Works from any machine with Docker + AWS CLI configured. No dependency on GitHub, GitLab, or any CI platform.

### Optional: GitHub Actions CI/CD

If using GitHub, `.github/workflows/deploy.yml` automates deploy on push to `main`. Requires OIDC setup (see workflow file). This is optional — the deploy script is the primary method.

## Auto-Scaling

Configured in CDK stack:

| Parameter | Value |
|-----------|-------|
| Min tasks | 2 |
| Max tasks | 10 |
| CPU target | 60% |
| Request target | 500 req/target |
| Scale-out cooldown | 60s |
| Scale-in cooldown | 300s |

Worker runs as a single task (no scaling needed — one worker is sufficient).

## Nginx Benefits

The production Dockerfile uses Nginx (via `Dockerfile.prod`):
- **Static files served directly** — CSS, JS, HTML served by Nginx at ~10x the speed of Flask/Gunicorn
- **Connection buffering** — Nginx absorbs slow clients, protecting Gunicorn workers
- **Gzip compression** — automated for text/JSON responses
- **Static asset caching** — 1-hour cache headers on CSS/JS/images
- **Health check endpoint** — `/health` proxied to `/api/stats`

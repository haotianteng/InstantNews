- Importing `app` (any submodule) triggers `app/__init__.py` side-effects that log `POLYGON_API_KEY not set — market data service disabled` to **stderr**. Safe: it does not pollute stdout, so stdout-capture tests stay clean.

### Stdout-capture test pattern (EMF / structured logs)
For modules that emit structured log lines via `print(..., flush=True)`, use pytest's `capfd` fixture (captures at the file-descriptor level, so it sees `flush=True` output correctly). Parse the captured string by splitting on newlines and ignoring blanks:

```python
def _parse_single_line(captured: str) -> dict:
    lines = [ln for ln in captured.splitlines() if ln.strip()]
    assert len(lines) == 1, f"expected 1 line, got {len(lines)}"
    return json.loads(lines[0])

def test_emit(capfd):
    emit_metric("NS", "M", 1.0, dimensions={"A": "b"})
    out, _err = capfd.readouterr()
    d = _parse_single_line(out)
    ...
```

`capfd.readouterr()` clears the buffer between calls, so multiple emissions in one test work if you read after each.

### EMF JSON shape (from live spec fetch 2026-04-17)
- `Dimensions` is a list of lists. Outer array has `minItems: 1` (spec-mandated) — even when emitting "no dimensions", we must render `Dimensions: [[]]` (one empty DimensionSet). This is valid because each DimensionSet has `minItems: 0`.
- Dimension *values* must be strings at the root level. Ints/floats/bools must be coerced (`str(value)`) before emission or CloudWatch drops the datapoint silently.
- `Timestamp` is `int(time.time() * 1000)` — integer ms since epoch.
- Metric values live at the **root** of the JSON (not inside `_aws`). `_aws.CloudWatchMetrics[0].Metrics` contains only `{Name, Unit}` definitions that *reference* the root values by name.
- One EMF log line = one timestamp = up to 100 metrics (AWS hard limit). Prefer `emit_metrics()` multi-metric variant over repeated `emit_metric()` calls to cut log volume.
- Serialize with `json.dumps(..., separators=(",",":"))` to keep lines compact.

### Shared helper location
`app/services/metrics.py` is stdlib-only (no boto3, no requests, no pydantic). Any later story (US-003, US-004, US-005) should import from there — do not re-implement the EMF schema.

### Learnings

- **EMF spec has a subtle trap**: the outer `Dimensions` array has `minItems: 1` but each inner `DimensionSet` has `minItems: 0`. That means "no dimensions" is `[[]]` NOT `[]`. Mis-reading this as `[]` produces a payload CloudWatch silently rejects (no metric ingested, no error surfaced to the producer).
- **Dimension values must be JSON strings**: CloudWatch treats non-string values as the datapoint being malformed. Coerce at the helper boundary, don't trust callers.
- **`print(..., flush=True)` is sufficient for ECS awslogs** — no need for `sys.stdout.write` + `sys.stdout.flush` pair, and no need for the Python logging stdlib. Simpler is better here because the helper is called from many layers.
- **`capfd` (not `capsys`) for FD-level capture**: a helper using `print(..., flush=True)` works with either, but `capfd` is the correct choice if a future helper ever calls `os.write(1, ...)` directly or if we test subprocess output. Picked `capfd` for forward-compatibility.

### Environment notes for Tester

- The happy-path test monkeypatches `source_poller.datetime` with a frozen subclass so the "now" used internally is deterministic. If you write your own validation script, note that `datetime.now(tz=...)` inside `_run_once` is called via the module-bound name, not the imported `datetime` — so monkeypatching at the module level works, but patching `datetime.datetime` globally won't.

### Learnings

- **ISO-8601 parser tolerance**: `datetime.fromisoformat` in Python 3.11 accepts `+00:00` suffix natively but NOT trailing `Z`. Every source in this repo emits `+00:00` via `feed_parser.utc_iso`, but any future source that formats with `Z` would silently drop out of the median. Explicit `if candidate.endswith("Z"): candidate = candidate[:-1] + "+00:00"` is the portable fix — Python 3.13 accepts Z natively but we can't rely on that yet.
- **Latency-of-nothing is undefined, not zero**: if a tick finds zero new rows, emitting `IngestLatencySeconds=0` pulls p50 downward and makes healthy-but-quiet sources look faster than genuinely fast ones. Skip the metric instead of emitting 0.

### Environment notes for Tester

- `_dispatcher_loop` is NOT covered by unit tests — it's a `while not _stop_event.is_set()` that depends on module-level globals (`_queue`, `_stop_event`). Exercising it cleanly requires subprocess or heavy monkeypatching. The post-deploy CloudWatch `QueueDepth` assertion is the cheaper path. If you want an in-process smoke, you can drive it by pre-seeding `_queue`, setting `_stop_event` just before calling `_dispatcher_loop(session_factory_stub, batch_window_seconds=0.05)`, and asserting `capfd` saw at least one QueueDepth line.
- `_call_model` tests patch `_get_minimax_client` / `_get_anthropic_client` / `_call_bedrock` at the module level. If you add your own validation, note that the MiniMax / Anthropic clients expose a `.messages.create(**kwargs)` surface that returns an object with `.content = [block, ...]` where each block has `.type == "text"` and `.text`. The test's `_FakeAnthropicClient` reproduces that shape.
- `analyze_articles_batch` bails early with `{}` when `BEDROCK_ENABLED=False` OR `articles=[]`. The metric emission happens only on the non-early-return path.

### Learnings

- **"Who served the request" vs "who was attempted"**: emitting `BackendChosen` only on the successful branch gives a clean fallback-chain donut. Emitting on every attempted branch (including failures) pollutes the donut with attempts that didn't actually serve traffic — you'd see MiniMax showing as the most popular backend even when it's failing 100% of the time and Claude is the true producer. The correct signal is served-not-tried.
- **QueueDepth rate-limiting via monotonic timestamp**: a `last_emit_mono` float + `time.monotonic() - last_emit_mono >= N` check is simpler than wrapping the emit in a ThreadPoolExecutor-based scheduler, and it's resistant to wall-clock jumps. 5 s cap on an otherwise-unbounded inner loop keeps the emission rate bounded even if `batch_window_seconds` is tuned down.

### Learnings

- **Header parse tolerance pattern**: helper returns `Optional[int]`; caller conditionally appends the metric to the payload only if the parse succeeded. Beats emitting `NaN`, `-1`, or `0` sentinel values that would bias CloudWatch aggregations.
- **Emit from the parse site, not the transport**: `_get(path, params)` is generic across endpoints whose rate-limit models differ (`/users/by/username` has different windows than `/tweets/search/recent`). Emitting inside `search_recent` — where we already parse the body — is both cheaper (no double-parse) and more correct (scoped to the endpoint whose semantics the dashboard cares about).
- **Pre-existing `import-untyped` for requests**: `requests` ships without type stubs in this environment. Rather than installing `types-requests` (adds a runtime dep that's global to the repo and could trigger other mypy issues), scope the ignore narrowly: `import requests  # type: ignore[import-untyped]`. This is the minimal change and is clearly tagged for future cleanup.

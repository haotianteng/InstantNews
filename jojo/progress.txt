### Stdout-capture test pattern (EMF / structured logs)
For modules that emit structured log lines via `print(..., flush=True)`, use pytest's `capfd` fixture (captures at the file-descriptor level, so it sees `flush=True` output correctly). Parse the captured string by splitting on newlines and ignoring blanks:

```python
def _parse_single_line(captured: str) -> dict:
    lines = [ln for ln in captured.splitlines() if ln.strip()]
    assert len(lines) == 1, f"expected 1 line, got {len(lines)}"
    return json.loads(lines[0])

def test_emit(capfd):
    emit_metric("NS", "M", 1.0, dimensions={"A": "b"})
    out, _err = capfd.readouterr()
    d = _parse_single_line(out)
    ...
```

`capfd.readouterr()` clears the buffer between calls, so multiple emissions in one test work if you read after each.

### EMF JSON shape (from live spec fetch 2026-04-17)
- `Dimensions` is a list of lists. Outer array has `minItems: 1` (spec-mandated) — even when emitting "no dimensions", we must render `Dimensions: [[]]` (one empty DimensionSet). This is valid because each DimensionSet has `minItems: 0`.
- Dimension *values* must be strings at the root level. Ints/floats/bools must be coerced (`str(value)`) before emission or CloudWatch drops the datapoint silently.
- `Timestamp` is `int(time.time() * 1000)` — integer ms since epoch.
- Metric values live at the **root** of the JSON (not inside `_aws`). `_aws.CloudWatchMetrics[0].Metrics` contains only `{Name, Unit}` definitions that *reference* the root values by name.
- One EMF log line = one timestamp = up to 100 metrics (AWS hard limit). Prefer `emit_metrics()` multi-metric variant over repeated `emit_metric()` calls to cut log volume.
- Serialize with `json.dumps(..., separators=(",",":"))` to keep lines compact.

### Shared helper location
`app/services/metrics.py` is stdlib-only (no boto3, no requests, no pydantic). Any later story (US-003, US-004, US-005) should import from there — do not re-implement the EMF schema.

### Learnings

- **EMF spec has a subtle trap**: the outer `Dimensions` array has `minItems: 1` but each inner `DimensionSet` has `minItems: 0`. That means "no dimensions" is `[[]]` NOT `[]`. ### Learnings

- **ISO-8601 parser tolerance**: `datetime.fromisoformat` in Python 3.11 accepts `+00:00` suffix natively but NOT trailing `Z`. - **Latency-of-nothing is undefined, not zero**: if a tick finds zero new rows, emitting `IngestLatencySeconds=0` pulls p50 downward and makes healthy-but-quiet sources look faster than genuinely fast ones. Skip the metric instead of emitting 0.

### Learnings

- **"Who served the request" vs "who was attempted"**: emitting `BackendChosen` only on the successful branch gives a clean fallback-chain donut. Emitting on every attempted branch (including failures) pollutes the donut with attempts that didn't actually serve traffic — you'd see MiniMax showing as the most popular backend even when it's failing 100% of the time and Claude is the true producer. The correct signal is served-not-tried.
- **QueueDepth rate-limiting via monotonic timestamp**: a `last_emit_mono` float + `time.monotonic() - last_emit_mono >= N` check is simpler than wrapping the emit in a ThreadPoolExecutor-based scheduler, and it's resistant to wall-clock jumps. 5 s cap on an otherwise-unbounded inner loop keeps the emission rate bounded even if `batch_window_seconds` is tuned down.

### Learnings

- **Cost Explorer DAILY granularity excludes `End`**: for a "last 7 days" view we ask `[today-7, today]` which yields 7 buckets ending yesterday. Asking for `[today-7, today+1]` to include today's partial-day bucket would return a too-low value AND pays for CE's $0.01 call regardless.

Two calls (totals + grouped) is the canonical way to populate a stacked bar chart without doing math on the frontend.
- **X API bearer token reuse**: the same `X_API_BEARER_TOKEN` secret that powers `twitter_source.py` polling also authorises `/2/usage/tweets`. No separate credential, no AWS Secrets Manager change.
- **Separate inner-vs-outer TTLs matter**: a single 1h cache on the whole response would cause a freshly-reset monthly quota (e.g. rolling past midnight on the 1st) to stay at "95% used" for up to an hour. The 5-min inner cache on the X portion caps that staleness window at 5 minutes while still massively reducing calls to the rate-limited `/2/usage/tweets` endpoint.

### Environment notes for future panel testers (US-009..US-013)

- **App JWT `sub` claim carries DB user_id (str), not firebase_uid**: `app/auth/middleware._try_app_jwt` does `int(claims['sub'])` then `session.query(User).filter_by(id=user_id)`. Mint tokens with `sub=str(user.id)` where `user.id` is the SQLite primary key row. The tester setup doc originally suggested firebase_uid — that would fail silently (token verifies, user_id=0, no match → anonymous). Correct pattern in `jojo/test_results/US-008/attempt-1/setup_admin.log`.
- **Playwright-MCP transport lacks `setExtraHTTPHeaders`**: you cannot attach `Authorization: Bearer <jwt>` to top-level navigations via the MCP. Two workarounds: (a) inject `signal_app_token` into localStorage before nav so `SignalAuth.fetch` picks it up on subsequent API calls (the panel JS only hits `/admin/api/metrics/...` from SignalAuth.fetch, not on navigation), then navigate to an unauth'd static mirror of the built HTML; (b) bypass browser entirely for panel-logic tests and use `fetch` via `browser_evaluate` after injecting the token. For US-008 the panel-less shell was auth-agnostic so we used (a) — panel stories hitting real admin APIs will need (a) with localStorage seeding.
- **auth.js cold-load 400 error suppression**: to keep `Zero console errors` assertion clean across *cold* navigations, pre-seed `localStorage.signal_auth_region='global'` + `signal_auth_region_ts=<now-ms>` before `browser_navigate`. Otherwise a single 400 from `https://www.googleapis.com/identitytoolkit/v3/relyingparty/getProjectConfig?key=test` will fire once per cold visit (shared auth.js behaviour, swallowed in try/catch, safe to ignore but visually noisy in console logs).
- **Static-server harness for browser-slice testing**: `cd static && python -m http.server 8766` works perfectly for any US-008/US-009+ test that needs to exercise the JS bundle without the auth-gated Flask route — the `static/admin-monitoring.html` + `/assets/admin-monitoring-*.js` bundle is identical whether Flask or the stdlib server delivers it.
- **Warm-up pattern**: the first Playwright navigation after clearing localStorage gets the 400 probe; subsequent navigations (with cached region) show 0 errors. If a panel test needs a cold run, warm the region first: navigate once, `browser_evaluate` to confirm `localStorage.signal_auth_region`, then proceed.

### Learnings

- **`@require_admin` on HTML GET returns 401 JSON, not a redirect**: correct behaviour for a VPN-gated admin tool, but means browser nav without a token shows a raw JSON error page rather than a sign-in prompt. For local testing this is a feature (easy assertion target); for real admin UX the frontend's `SignalAuth.onAuthChange` listener triggers a Firebase popup on the next load.
- **Dual-source selector verification is cheap and catches build-vs-source drift**: grepping the five required selectors against both `frontend/admin-monitoring.html` (source) AND the authed `curl` response body (build + Flask serve path) guarantees you're asserting against the actual bytes the browser receives, not just the source-of-truth template. - **Playwright MCP + cold auth.js = spurious 1-time 400**: don't conflate the region-probe 400 with real page errors. A quick check is to reload the page (warm cache) and re-capture console — if the error disappears, it was the region probe. Documented in `test_report` as not-a-defect.

### Learnings

- **`registerPanel(name, fn)` fires `fn` synchronously on registration** (by design — late-joining panels still get data). That means on first page load, panels kick off a fetch before the user interacts at all. For test mocking this is slightly awkward: mocks must be installed before `setRange` runs. Workaround is to wait for `window.__monitoring__` to exist, then overwrite `fetchMetrics`/`fetchCost`, then call `refresh<Panel>(range)` to force a re-render.
- **CE GroupBy=SERVICE is summed across the range, not per-day**, so the "stacked bar" chart is an approximation — each day's total is sliced by the across-range service mix. Honest per-day-per-service would need a second CE query with `Granularity=DAILY + GroupBy=SERVICE` and per-day service lookup. For MVP the approximation is fine; documented in code.
- **CloudWatch stat strings accept `p50`/`p95` directly** — no need to pre-convert to `ExtendedStatistics`. `MetricDataQuery.MetricStat.Stat` takes "p50" as a string; this matched our admin API passthrough which forwards `q.stat` unchanged.
- **SVG donut with concentric circles** is simpler than `<path>` arcs: three `<circle>` with `stroke-dasharray="<pct> <100-pct>"` + cumulative `stroke-dashoffset` gives a flawless donut without arc-math bugs. Cheaper than a chart library.
- **`CSS.escape()`** is essential when using `data-source` as a selector — source names like `Twitter/@POTUS` contain `/` and `@` which are valid CSS selector metachars.
